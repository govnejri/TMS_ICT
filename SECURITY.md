# Безопасность

## Сообщение об уязвимостях безопасности

Мы серьезно относимся к безопасности нашего проекта. Если вы обнаружили уязвимость, пожалуйста, сообщите нам об этом ответственно.

### Как сообщить

**НЕ** создавайте публичные GitHub Issues для уязвимостей безопасности.

Вместо этого, пожалуйста:

1. Отправьте email на: **security@example.com** (замените на ваш email)
2. Или используйте [GitHub Security Advisories](https://github.com/ваш-username/ict/security/advisories/new)

### Что включить в отчет

Пожалуйста, предоставьте:

- Описание уязвимости
- Шаги для воспроизведения
- Возможное влияние
- Предложения по исправлению (если есть)
- Ваши контактные данные для дальнейшей связи

### Время ответа

- Мы подтвердим получение в течение **48 часов**
- Предоставим более подробный ответ в течение **7 дней**
- Будем держать вас в курсе о процессе исправления

### Процесс раскрытия

1. Получение отчета
2. Подтверждение уязвимости
3. Разработка исправления
4. Тестирование
5. Выпуск обновления
6. Публичное раскрытие (после исправления)

## Лучшие практики безопасности

### Для разработчиков

- ✅ Никогда не коммитьте `.env` файлы
- ✅ Используйте переменные окружения для секретов
- ✅ Регулярно обновляйте зависимости
- ✅ Проверяйте код на уязвимости (`npm audit`)
- ✅ Используйте HTTPS для всех API запросов
- ✅ Валидируйте все пользовательские данные
- ✅ Используйте prepared statements для SQL запросов

### Для пользователей

- ✅ Используйте сильные пароли (когда аутентификация будет реализована)
- ✅ Не делитесь API ключами
- ✅ Регулярно проверяйте активность аккаунта
- ✅ Обновляйте приложение до последних версий

## Известные ограничения

На текущий момент (v1.0.0):

- Отсутствует аутентификация пользователей
- API ключи хранятся на клиенте (VITE_* переменные)
- Нет rate limiting для API запросов

Эти ограничения будут устранены в будущих версиях.

## Зависимости безопасности

Регулярно проверяйте и обновляйте зависимости:

```bash
# Проверить уязвимости
npm audit

# Автоматически исправить (где возможно)
npm audit fix

# Обновить все зависимости
npm update
```

## Настройки безопасности

### Content Security Policy

Рекомендуем добавить CSP заголовки в production:

```html
<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; 
               script-src 'self' 'unsafe-inline'; 
               style-src 'self' 'unsafe-inline'; 
               img-src 'self' data: https:; 
               connect-src 'self' https://api.supabase.co https://api.shipengine.com https://nominatim.openstreetmap.org;">
```

### CORS настройки

Убедитесь, что CORS настроены правильно в Supabase и других API.
